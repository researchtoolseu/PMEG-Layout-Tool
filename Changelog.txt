Changelog
This page lists all updates released for the PMEG Fenestration Layout Tool.

*** April 19, 2026 - Version 2.18 (Dual-report system & clinical report refinement)


NEW FEATURES
- Introduced dual-report output:
  - PMEG_LAYOUT_REPORT.txt → Clinical report (user-friendly)
  - PMEG_LAYOUT_REPORT_TECHNICAL.txt → Technical report (full metadata & internal values)
  - PMEG_LAYOUT_REPORT.csv → Structured machine-readable export

- Clinical report redesigned for improved readability:
  - Structured sections (Patient, Planning Parameters, Version Notes, Fenestrations, Distances)
  - Removal of internal/debug metadata from main report
  - Cleaner terminology and formatting

- Refined Fenestration Summary (clinical report):
  - Aligned tabular layout
  - Improved column naming (Angle, Diameter, y position, x position)
  - Consistent rounding and spacing
  - Optimized for quick clinical interpretation

- Version notes integration improved:
  - Displayed once in a dedicated section (no duplication)
  - Included in both TXT and CSV outputs (clean handling)

TECHNICAL IMPROVEMENTS
- Separation of report logic into clinical vs technical outputs
- Full metadata preserved in technical report
- Cleaner CSV export structure with explicit version_notes handling

CLINICAL / PRACTICAL IMPROVEMENTS
- Significantly improved report readability for clinical use
- Better suitability for:
  - case documentation
  - teaching material
  - scientific communication
- Maintains full traceability through technical report

NOTES
- No changes to core PMEG geometry or plotting logic
- Fully backward compatible with previous input formats

*** April 18, 2026 - Version 2.17 (Version notes integration & workflow enhancement)


NEW FEATURES
- Added version-specific notes (version_notes)
  - Supports multiline free-text input (Excel / CSV)
  - Designed to capture operator planning thoughts per case version
  - Separate from fenestration-level notes

- Added “Version notes” box in MAIN PDF
  - Wrapped, formatted text block
  - Positioned separately from fenestration table
  - Handles line breaks and long text (with truncation if needed)

- Included version notes in reports
  - Exported to:
    - PMEG_LAYOUT_REPORT.txt
    - PMEG_LAYOUT_REPORT.csv
  - Preserved as part of case-version documentation

EXCEL WORKFLOW IMPROVEMENTS
- Multiline input supported in PMEG_Input.xlsm (Alt+Enter)
- Automatic clearing of version notes after successful run
  - Prevents carry-over between runs
  - Maintains clean input for iterative planning

CLINICAL / PRACTICAL IMPROVEMENTS
- Adds traceable decision context per version
- Improves reproducibility and auditability
- Connects geometric planning with clinical reasoning

*** April 17, 2026 - Version 2.16 (Print Safety and Film Identification Improvements)

CLINICAL / PRACTICAL IMPROVEMENTS
- Added a print-safety notice inside the 100 mm calibration square in the MAIN PDF layout:
  "PRINT AT 100% (ACTUAL SIZE) / CHECK SCALE BEFORE USE"
- Added vertical patient identification text in the transparent FILM PDF cut margins, including:
  patient name, age, study date, and instance ID (patient's version and ID)
- Patient identification is printed on both cut margins of the FILM layout to reduce the risk of confusion when multiple patient layouts are present or when the film is rolled

WORKFLOW / SAFETY BENEFITS
- Reinforces mandatory scale verification before use
- Improves traceability of printed transparent layouts
- Reduces wrong-patient / wrong-version handling risk in multi-case environments

*** April 11, 2026 – Version 2.14 (Film Boundary & Extended Distal Measurements)

NEW FEATURES
- Transparent film boundary visualization (MAIN output):
  A horizontal reference line is now added at the level of film_height_mm, clearly indicating the bottom edge of the transparent film in both PDF and PNG outputs
- Extended longitudinal measurements to film end:
  The vertical measurement scales have been enhanced to include:
  - Distance from the center of the last (most distal) fenestration to the bottom end of the transparent film
  - Distance from the bottom edge of the last fenestration to the bottom end of the transparent film

IMPROVEMENTS
- Improved intraoperative usability by explicitly visualizing the available distal working length of the graft relative to the transparent film
- Enhanced measurement completeness by ensuring that all distal clearances are quantified, not only inter-fenestration distances

CLARIFICATIONS
- proximal_cover_mm defines the distance from the top of the graft (transparent film) to the top of the anchor fenestration
- The bottom of the anchor fenestration remains the ZERO reference point for all longitudinal measurements

CLINICAL / PRACTICAL IMPROVEMENTS
- Enables accurate planning of layouts with fenestrations located at the distal end of the graft, such as with a thoracic graft interfering distally with CA or SMA

*** April 2, 2026 - Version 2.12 (Configurable Reduction Tie Positions)

NEW FEATURES
- Script renamed to "pmeg_layout_tool_v2.12.py" for standardization
- Configurable Reduction Tie Positions:
  - Added support for defining reduction tie clock positions via metadata:
    # tie_positions_clock: 5,6,7
  - Default remains 4,6,8 if not specified
  - Allows patient-specific customization of graft planning strategy
  - Supports any valid clock positions (1–12)

- Dynamic Tie Position Mapping:
  - Clock positions are now internally converted to angular coordinates
  - Enables flexible placement beyond fixed predefined locations

- Validation of Tie Positions:
  - Ensures exactly 3 positions are provided
  - Validates values are within 1–12
  - Prevents duplicate entries

MINOR CHANGES
- Refactored reduction tie drawing logic to remove hard-coded positions (4,6,8)
- Improved modularity of tie guide generation for future extensions
- Tie positions are now included in the generated REPORT output

CLINICAL / PRACTICAL IMPROVEMENTS
- Enables tailoring of reduction ties based on:
  - Patient anatomy
  - Graft configuration
  - Operator preference
- Improves intraoperative alignment flexibility

NOTES
- 6 o’clock position continues to be represented at both wrap edges of the unrolled graft
- Backward compatibility maintained (older input files still work without modification)
