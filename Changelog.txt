Changelog
This page lists all updates released for the PMEG Fenestration Layout Tool.


-------------------------------------------------------------------------------

*** May 31, 2026 - Version 2.23 (Adding zero reducing ties)

NEW FEATURES
- Added support for `tie_num_rows = 0`.
- When `tie_num_rows` is set to `0`, no reducing tie guide lines or dots are drawn.
- `tie_num_rows = 0` overrides any value entered in `tie_positions_clock`.
- Added support for “none-style” values in `tie_positions_clock`, including:
  - `None`
  - `No`
  - `Off`
  - `0`
- Improved Excel input flexibility for cases where no reducing ties are planned.
- Clinical/report output now displays tie positions as `None` when reducing ties are disabled.

-------------------------------------------------------------------------------

*** May 30, 2026 - Version 2.22 

NEW FEATURES
- Updated accepted/user-facing values for `orientation`:
  - `Portrait`
  - `Landscape`
- Updated accepted/user-facing values for `circumferential_mode`:
  - `Nominal`
  - `Constrained`
- Maintained backward compatibility with previous lowercase values:
  - `portrait`
  - `landscape`
  - `nominal`
  - `constrained`
- Modified `tie_positions_clock` behavior:
  - blank cell / blank metadata value now disables reducing tie guide drawing.
- Allowed generation of layouts without reducing tie position guides.

-------------------------------------------------------------------------------

*** May 30, 2026 - Version 2.21 (Circumeferential mode)

MAJOR UPDATE

NEW FEATURES
- Added constrained circumferential mapping mode.
- Introduced `circumferential_mode` metadata option:
  - `nominal`
  - `constrained`
- Added support for `native_aorta_diam_mm` / equivalent mapping diameter metadata in constrained mode.
- In constrained mode, fenestration x-positions are calculated using the native aortic diameter.
- Graft wrap edges, cut lines, and nominal graft perimeter remain based on the nominal graft diameter.
- Added reporting of circumferential mode and θ→x mapping diameter in layout tables and generated reports.
- Preserved standard nominal graft-based mapping as the default behavior.

*** April 27, 2026 - Version 2.20 (Planning Mode Logic & XLSM Integration)


MAJOR UPDATE

NEW FEATURES
- Introduced explicit metadata field: planning_mode (Planning / Legacy)
- Dual-mode workflow supported within the same file:
  - Planning mode → uses dist_from_zero_mm + proximal_cover_mm + anchor
  - Legacy mode → uses y_mm
- Script now ignores inactive input fields instead of requiring them to be empty
  (allows safe coexistence of both planning approaches in the same dataset)

VALIDATION & SAFETY IMPROVEMENTS
- Mode-dependent validation:
  - Planning mode requires dist_from_zero_mm and proximal_cover_mm
  - Legacy mode requires y_mm
- Removed strict requirement for unused fields to be blank (reduces user error risk)
- Improved error messages for missing required inputs

REPORTING IMPROVEMENTS
- Added planning mode display in:
  - Clinical TXT report
  - Technical TXT report
  - Main PDF (Fenestrations table as “Mode: Planning / Legacy”)
- Enhances traceability and reproducibility of planning approach

XLSM TEMPLATE UPDATES (v2.20)
- Added new metadata field: planning_mode
- Added Definition column (clear, one-line description of each metadata field)
- Added Guidance column for user instructions
- Improved structure: Definition | Metadata | Value | Guidance
- Introduced visual logic (grey-out concept) for inactive fields depending on planning mode
- Updated top instruction line for clearer workflow guidance

USABILITY IMPROVEMENTS
- Allows users to switch between Planning and Legacy modes without deleting previous data
- Enables parallel storage of both measurement strategies for comparison and verification
- Reduced risk of data loss and re-entry errors

INTERNAL CHANGES
- Refactored planning-mode detection logic to support explicit mode selection
- Improved metadata handling and normalization
- Minor stability and robustness improvements

NOTES
- Greyed-out fields in XLSM indicate inactive inputs but are not required to be empty
- Planning mode is recommended for new cases; Legacy mode maintained for compatibility

*** April 22, 2026 - Version 2.19 (Enhanced Longitudinal Scale Completeness)


NEW FEATURES
- Added proximal center-to-center distance
  • The center-to-center scale now includes:
    Top of graft (y = 0) → center of the most proximal fenestration
  • Completes the longitudinal reference system for improved interpretability

IMPROVEMENTS
- Symmetry between longitudinal scales
  • Center-to-center and bottom-to-bottom scales now follow the same structure:
    - Proximal segment
    - Intermediate (fenestration-to-fenestration) segments
    - Distal segment to film end

- Improved visual consistency
  • Better alignment of dimension annotations across the full graft length

CLINICAL / PRACTICAL IMPACT
- Facilitates rapid understanding of absolute fenestration positioning
- Reduces need for manual estimation of proximal distances
- Improves intraoperative orientation and planning confidence

*** April 11, 2026 – Version 2.14 (Film Boundary & Extended Distal Measurements)

NEW FEATURES
- Transparent film boundary visualization (MAIN output):
  A horizontal reference line is now added at the level of film_height_mm, clearly indicating the bottom edge of the transparent film in both PDF and PNG outputs
- Extended longitudinal measurements to film end:
  The vertical measurement scales have been enhanced to include:
  - Distance from the center of the last (most distal) fenestration to the bottom end of the transparent film
  - Distance from the bottom edge of the last fenestration to the bottom end of the transparent film

IMPROVEMENTS
- Improved intraoperative usability by explicitly visualizing the available distal working length of the graft relative to the transparent film
- Enhanced measurement completeness by ensuring that all distal clearances are quantified, not only inter-fenestration distances

CLARIFICATIONS
- proximal_cover_mm defines the distance from the top of the graft (transparent film) to the top of the anchor fenestration
- The bottom of the anchor fenestration remains the ZERO reference point for all longitudinal measurements

CLINICAL / PRACTICAL IMPROVEMENTS
- Enables accurate planning of layouts with fenestrations located at the distal end of the graft, such as with a thoracic graft interfering distally with CA or SMA

*** April 2, 2026 - Version 2.12 (Configurable Reduction Tie Positions)

NEW FEATURES
- Script renamed to "pmeg_layout_tool_v2.12.py" for standardization
- Configurable Reduction Tie Positions:
  - Added support for defining reduction tie clock positions via metadata:
    # tie_positions_clock: 5,6,7
  - Default remains 4,6,8 if not specified
  - Allows patient-specific customization of graft planning strategy
  - Supports any valid clock positions (1–12)

- Dynamic Tie Position Mapping:
  - Clock positions are now internally converted to angular coordinates
  - Enables flexible placement beyond fixed predefined locations

- Validation of Tie Positions:
  - Ensures exactly 3 positions are provided
  - Validates values are within 1–12
  - Prevents duplicate entries

MINOR CHANGES
- Refactored reduction tie drawing logic to remove hard-coded positions (4,6,8)
- Improved modularity of tie guide generation for future extensions
- Tie positions are now included in the generated REPORT output

CLINICAL / PRACTICAL IMPROVEMENTS
- Enables tailoring of reduction ties based on:
  - Patient anatomy
  - Graft configuration
  - Operator preference
- Improves intraoperative alignment flexibility

NOTES
- 6 o’clock position continues to be represented at both wrap edges of the unrolled graft
- Backward compatibility maintained (older input files still work without modification)
